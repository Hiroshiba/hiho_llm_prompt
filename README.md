## wikipedia の情報を集める

```cline
`VOICEVOX`に関するwikipediaの情報をcurlと`https://ja.wikipedia.org/w/rest.php/v1/page/`APIで取得し、内容を５行で要約してください。
要約したものを`hiho_docs/wikipedia_summary.md`に保存してください。
装飾等は不要です。
wikipediaページが存在しない場合は何もしないでください。
```

## 最初の要約

```bash
bash 1_summary.bash
```

## ファイルツリーとそれぞれの要約

```bash
bash 2_filetree.bash
```

```cline
以下の手順に従って、`hiho_docs/filetree.md`にファイルごとの要約情報を追加してください。
1. イントロダクション`hiho_docs/introduction.md`を読んでください
2. `hiho_docs/filetree.txt`内のまだ要約されていない行を選んでください
3. その行のパスがディレクトリの場合は最後に回して手順2に戻ってください
4. その行のパスのファイルを読んでください
5. そのファイルの内容を１行で要約してください。簡単なものは一言だけ、複雑なものは簡単な説明の後、要点を書いてください。
6. 要約した内容を、`hiho_docs/filetree.txt`のその行最後にスペース` `を追加したあと、後ろに追記してください
7. その行が最後の行でない場合は手順2に戻ってください
8. まだ要約されていない行を選び、そのディレクトリを要約してください
9. そのディレクトリの内容を１行で要約してください。簡単なものは一言だけ、複雑なものは簡単な説明の後、要点を書いてください。
10. 要約した内容を、`hiho_docs/filetree.txt`のその行最後にスペース` `を追加したあと、後ろに追記してください
11. その行が最後の行でない場合は手順8に戻ってください

以下注意事項です。
* パスの最初にインデントスペースがある場合、それはその行より上で同じ数のインデントスペースが無いディレクトリの下にあるファイルです
  * 例えば`.github`の下に`  workflows`があり、その下に`    test.yml`があった場合のパスは`.github/workflows/test.yml`です
* よくわからなかったときは後回しにしてください
* 10つ要約するたびにイントロダクション`hiho_docs/introduction.md`を読み直してください
* 日本語で要約してください
```
